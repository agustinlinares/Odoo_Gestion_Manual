Instalación de Odoo 18 en Ubuntu 24.04.3 LTS

Procedimiento paso a paso para desplegar Odoo 18 (Community) en Linux Ubuntu: dependencias, PostgreSQL, entorno virtual, configuración y puesta en marcha.

    0. Recomendaciones previas

    Antes de empezar: trabaja en una máquina actualizada, con acceso a Internet y permisos de administrador. Este manual asume una instalación limpia de Ubuntu 24.04.3 LTS.

    Nota importante: en varios comandos se usa la contraseña admin como ejemplo (por rapidez). En un entorno real, usa contraseñas fuertes y guarda el valor de admin_passwd (master password) en un gestor seguro.

    Diagrama de arquitectura (Apache ↔ Odoo ↔ PostgreSQL)

    En producción se recomienda acceder a Odoo mediante un proxy inverso (Apache) en los puertos 80/443, mientras que Odoo escucha internamente en 8069 y se conecta a PostgreSQL en 5432.

    Nota: si vas a usar longpolling/WebSocket, configura el proxy como se detalla en el manual de Apache.

    1. Actualizar sistema e instalar dependencias

    Actualiza paquetes y añade librerías necesarias para compilar dependencias de Python, conectar con PostgreSQL y generar assets.

    Terminal · dependencias
    sudo apt update && sudo apt upgrade -y

sudo apt install -y \
  git python3-pip python3-dev python3-venv \
  build-essential libssl-dev libffi-dev \
  libxml2-dev libxslt1-dev zlib1g-dev \
  libsasl2-dev libldap2-dev \
  libjpeg-dev libjpeg8-dev liblcms2-dev \
  libblas-dev libatlas-base-dev \
  libpq-dev

    (Opcional) Comprueba versión de Python:

    Terminal · python
    python3 --version

    2. Instalar y configurar PostgreSQL

    Odoo usa PostgreSQL como base de datos. Instálalo y crea un usuario de base de datos para Odoo.

    Terminal · PostgreSQL
    sudo apt install -y postgresql postgresql-client

sudo su - postgres -c "createuser -s odoo18"

    Ahora asigna contraseña al usuario odoo18 en PostgreSQL:

    Terminal · psql
    sudo su - postgres
psql

ALTER ROLE odoo18 WITH PASSWORD 'admin';
\q
exit

    3. Crear usuario del sistema para Odoo

    Por seguridad, Odoo no debería ejecutarse como root. Crearemos un usuario de sistema llamado odoo18 con su “home” en /opt/odoo18.

    Terminal · usuario
    sudo useradd -m -d /opt/odoo18 -U -r -s /bin/bash odoo18

    4. Clonar Odoo 18 y preparar entorno virtual

    Descarga el código de Odoo 18 desde GitHub y crea un entorno virtual para aislar dependencias.

    Terminal · clone + venv
    sudo su - odoo18
cd /opt/odoo18

git clone https://github.com/odoo/odoo.git --depth 1 --branch 18.0 --single-branch odoo

python3 -m venv venv
source venv/bin/activate
    Tip: si cierras la sesión y necesitas volver a entrar, repite: sudo su - odoo18 y luego source /opt/odoo18/venv/bin/activate.

    5. Instalar requisitos de Python

    Con el entorno virtual activado, instala los requisitos de Odoo. Esto puede tardar un poco dependiendo del equipo.

    Terminal · pip requirements
    pip install wheel
pip install -r /opt/odoo18/odoo/requirements.txt

    6. Configurar Odoo

    Crea un archivo de configuración para centralizar parámetros (usuario de base de datos, rutas de addons, logs, etc.). Dos opciones habituales:

    • Opción A (recomendada): config en /etc/odoo18.conf (modo “servidor”).
    • Opción B: config local en /opt/odoo18/odoo/debian/odoo.conf (modo “desarrollo”).

    Opción A — /etc/odoo18.conf

    Terminal · config
    sudo nano /etc/odoo18.conf

    Pega dentro:

    [options]
admin_passwd = admin
db_host = False
db_port = False
db_user = odoo18
db_password = admin

addons_path = /opt/odoo18/odoo/addons
logfile = /var/log/odoo/odoo18.log
    Terminal · logs
    sudo mkdir -p /var/log/odoo
sudo chown odoo18:odoo18 /var/log/odoo

    Opción B — debian/odoo.conf (si sigues un flujo similar al de tu guía)

    Terminal · debian/odoo.conf
    cd /opt/odoo18/odoo/debian
nano odoo.conf
    db_user = odoo18
db_password = admin
addons_path = /opt/odoo18/odoo/addons

    7. Arrancar Odoo (modo consola)

    Arranque directo para comprobar que todo funciona (ideal para pruebas).

    Terminal · run
    sudo su - odoo18
source /opt/odoo18/venv/bin/activate

cd /opt/odoo18/odoo
python3 odoo-bin -c /etc/odoo18.conf
    Si estás usando la opción B (config dentro del repo), el comando sería: python3 odoo-bin -c debian/odoo.conf.

    8. (Opcional recomendado) Crear servicio systemd

    Para producción o servidores, conviene crear un servicio para que Odoo arranque automáticamente y quede en segundo plano.

    Terminal · systemd
    sudo nano /etc/systemd/system/odoo18.service

    Contenido sugerido:

    [Unit]
Description=Odoo 18
After=network.target postgresql.service

[Service]
Type=simple
User=odoo18
Group=odoo18
SyslogIdentifier=odoo18
PermissionsStartOnly=true
ExecStart=/opt/odoo18/venv/bin/python3 /opt/odoo18/odoo/odoo-bin -c /etc/odoo18.conf
StandardOutput=journal+console

[Install]
WantedBy=multi-user.target

    Activa y arranca:

    Terminal · enable/start
    sudo systemctl daemon-reload
sudo systemctl enable odoo18
sudo systemctl start odoo18
sudo systemctl status odoo18

    9. Crear la base de datos desde el navegador

    Abre en el navegador: http://localhost:8069 (o http://IP_DEL_SERVIDOR:8069). Verás el selector de bases de datos (/web/database/selector). 

    Master password: admin
Database name: odoo18
Email: admin
Password: admin
Language: Spanish
Country: Spain
Demo Data: (desmarcado)
    ¡Listo! Si Odoo arranca sin errores, ya puedes instalar aplicaciones y empezar a usar tu módulo.

    10. Verificación final (servicio, puerto y logs)

    Con el servicio habilitado, valida que Odoo está activo, que escucha en el puerto 8069 y que no hay errores en el journal.

    Producción: no expongas :8069 directamente en Internet. Usa un proxy inverso con HTTPS y cabeceras correctas y activa proxy_mode = True en Odoo.
